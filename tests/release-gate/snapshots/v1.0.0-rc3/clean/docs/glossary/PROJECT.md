# Project Glossary (binding, project-specific)

Project-specific terms: internal nicknames, customer-domain jargon,
vendor / platform / site-specific shorthand, acronyms used inside this
project that are not covered by `docs/glossary/ENGINEERING.md`.

Rules:

- If a term is generic software-engineering vocabulary, it belongs in
  `ENGINEERING.md`, not here.
- If a term is customer-domain ground truth, mirror it into
  `CUSTOMER_NOTES.md` (verbatim from the customer) and cite the
  customer-notes row from the entry here.
- Ambiguities get resolved by `researcher` + `tech-lead` + the relevant
  `sme-<domain>` agent, then written here. Do not use a term
  differently in practice.
- One-line definition, then optional notes / source. Keep entries
  alphabetized within each section.
