# Allowlist for the `advisory-pointers` sub-gate (FR-006). Each non-comment
# non-empty line is a path that the scanner finds inside operator-facing
# strings but that legitimately does NOT exist in the candidate tree.
# Reasons fall into two classes:
#
#   1. Paths CREATED by the script at runtime in downstream projects (e.g.,
#      docs/intake-log.md is seeded by scripts/upgrade.sh into the project,
#      not into the template).
#   2. Example / placeholder filenames inside doc-comment narratives (e.g.,
#      "merge docs/PART_A.md and docs/PART_B.md into docs/COMBINED.md").
#
# Format: one path per line. Comments start with `#`.
#
# Maintenance: add a row when a new legitimate dangling reference appears;
# delete the row when the path becomes real in-tree. The gate trusts the
# allowlist contents; review changes via code-reviewer.
