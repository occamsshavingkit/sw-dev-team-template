# Examples

Fully-filled-in example projects, one per subdirectory. These are
**not** shipped to downstream projects (`scripts/scaffold.sh`
excludes `examples/`). They live in the template repo as reference
material — when you are unsure how a register or PM artifact should
look, look here.

| Subdirectory | Source | What it shows |
|---|---|---|
| `brewday-log-annotator/` | v0.1.0 dry-run (simulated customer Alex Keller) | Step 2 scoping filled into `OPEN_QUESTIONS.md` and `CUSTOMER_NOTES.md` end-to-end; Step 3 classical-composers naming applied; filled `docs/pm/CHARTER.md` and `docs/pm/LESSONS.md`. |

## Adding a new example

If a downstream project produces a notable filled-in artifact set
(project charter with a distinctive structure, an AGENT_NAMES mapping
from a category this repo has not demonstrated, a PMBOK risk register
that illustrates a domain-specific pattern), copy it here under a
descriptive subdirectory, redact any customer-sensitive content, and
add a row to the table above.

Examples are local-reference; they carry no binding weight on future
releases.
