## Detailed Findings

### 1. SemVer Tag Sorting Across Decade Boundaries (Issue #260)
* **Status**: Clean / Robust
* **Severity**: Met (Resolution Verified)
* **Audit Notes**: In `scripts/lib/semver.sh`, the legacy alphanumeric tag-sorting logic previously used a digit-count key suffix `sprintf("%010d", length(num)) "." num`. While this sorted tags with different digit counts (e.g. `rc9` vs `rc10`), it introduced sorting failures for tags with identical digit counts but different values (e.g. comparing `rc10` and `rc14` with `rc9`). The fix correctly replaces this with a direct numeric evaluation using `sprintf("%010d", num+0)`. This zero-pads the actual numeric suffix value to 10 digits (e.g., `0000000009` vs `0000000014`), which guarantees flawless numeric sort order across decade boundaries.
* **Traceability**: Sourced and validated via `tests/upgrade/test-version-check.sh` (tests cross-decade check `rc9 < rc10 < rc14`) and `scripts/upgrade.sh --self-test-semver` (both pass cleanly).

### 3. Agent Contract Schema Backfill & Insertion Ordering (Issues #261, #266, #267)
* **Status**: Clean / Robust
* **Severity**: Met (Resolution Verified)
* **Audit Notes**: The rc14 contract schema requires `hard_rules` and `output_format` sections on all canonical agents. Under `migrations/v1.0.0-rc14.sh`, preserved canonical agent files are automatically backfilled by extracting missing sections from the shipped rc14 upstream in `WORKDIR_NEW`, falling back to safe `TODO` placeholders if upstream is unreachable.
  * **Exclusion Safeguard**: SME files (`sme-*.md`, `sme-template.md`) and local agent supplements (`*-local.md`) are properly excluded from backfilling per customer rulings.
  * **Order-Preservation Resolution**: Fixed a critical ordering bug (issue #267) where running multiple section insertions sequentially on the same anchor line (the `Escalation` section) reversed their order in the final file (producing `Output format` before `Hard rules`). The migration now concatenates the missing blocks into a single combined payload and performs a single insertion, ensuring the canonical order `Escalation → Hard rules → Output format` is rigidly preserved.
  * Same-anchoring fixes were correctly back-propagated to `migrations/v1.0.0-rc9.sh` for idempotency.
* **Traceability**: Sourced and validated via `tests/upgrade/test-rc14-pass2-backfill.sh` (38/38 assertions passed cleanly) and `scripts/lint-agent-contracts.sh` (0 errors, 0 warnings).

* **Severity**: Met (Resolution Verified)
