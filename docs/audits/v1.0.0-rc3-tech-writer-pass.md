## Per-category findings

### Heading-consistency / agent-contract style drift

The most serious finding of this pass. Nine of the fourteen
agent contracts have a `<!-- TOC -->` block placed **before** the
YAML frontmatter (`---`-delimited block with `name:`, `tools:`, etc).

In Claude Code's agent-loader, the YAML frontmatter must be the first
non-blank thing in the file — that is how `name`, `tools`, and `model`
are parsed and how the agent registry indexes the file. A markdown
comment + heading list ahead of the frontmatter is likely to be
treated as body content and the frontmatter stops being recognised
as frontmatter. (Verification owed: a quick dispatch test against
one of the affected agents would confirm whether the loader is
tolerant or strict; this finding presumes strict semantics, which is
the safe default.)

Files affected (TOC ahead of frontmatter):

| Agent file | First non-blank line |
|---|---|
| `.claude/agents/tech-lead.md`            | `<!-- TOC -->` |
| `.claude/agents/architect.md`            | `<!-- TOC -->` |
| `.claude/agents/process-auditor.md`      | `<!-- TOC -->` |
| `.claude/agents/researcher.md`           | `<!-- TOC -->` |
| `.claude/agents/qa-engineer.md`          | `<!-- TOC -->` |
| `.claude/agents/security-engineer.md`    | `<!-- TOC -->` |
| `.claude/agents/onboarding-auditor.md`   | `<!-- TOC -->` |
| `.claude/agents/project-manager.md`      | `<!-- TOC -->` |
| `.claude/agents/sme-template.md`         | `<!-- TOC -->` |

Files unaffected (frontmatter first, TOC absent or below body):

| Agent file | First non-blank line |
|---|---|
| `.claude/agents/code-reviewer.md`        | `---` (frontmatter) |
| `.claude/agents/software-engineer.md`    | `---` (frontmatter) |
| `.claude/agents/sre.md`                  | `---` (frontmatter) |
| `.claude/agents/tech-writer.md`          | `---` (frontmatter) |
| `.claude/agents/release-engineer.md`     | `---` (frontmatter) |

**Likely root cause.** `scripts/gen-toc.sh` (commit ffd8c3e in the
prior session) inserted `<!-- TOC -->` blocks at the top of every
binding doc ≥ 100 lines without special-casing files whose first
content line is YAML frontmatter. The script presumably checks for
an H1; finding none (frontmatter is not an H1), it inserts the TOC
at line 1, displacing the frontmatter.

| ID | File | Lines | Issue | Recommendation | Severity |
|---|---|---|---|---|---|
| TW-001 | 9 agent files (above) | 1–N | TOC inserted ahead of YAML frontmatter; agent registry may fail to parse `name:` / `tools:` / `model:`. | (a) Dispatch test ONE of the affected agents to confirm whether the loader is strict. (b) If strict: move the TOC to immediately after the frontmatter on every affected file. (c) Patch `scripts/gen-toc.sh` to detect a leading `---` block and insert TOC after the closing `---`. | **blocker** if the loader is strict; **major** if tolerant (still a contract style break). |

Routing: `software-engineer` to fix `scripts/gen-toc.sh` and
re-run; `code-reviewer` to gate; `tech-lead` to ratify.
