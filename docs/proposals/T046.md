## 8. Duel

### 8.1 Findings (`qa-engineer`)

- **F-1 [Major]:** If the workflow assumes `scripts/lint-agent-contracts.sh` exists or has stable exit semantics without a preflight check, CI can fail every PR or, worse, pass while not exercising the intended contract validation; the failure is false confidence in required agent-contract gating.
- **F-2 [Major]:** If path filters are added too narrowly around `.claude/agents/*.md`, changes to schemas, routing docs, runtime adapters, or model-routing guidelines can skip the check; the failure is contract drift entering through untested dependency files.
- **F-3 [Critical]:** If the workflow uses `pull_request_target`, secrets, or broad write permissions for a lint-only check, untrusted PR content can execute with elevated token scope; the failure is CI supply-chain exposure unrelated to the contract being validated.

### 8.2 Rebuttals / revisions (`software-engineer`)

- F-1 addressed: `.github/workflows/agent-contract-check.yml` preflights `test -x scripts/lint-agent-contracts.sh` before invoking the script.
- F-2 addressed: the workflow uses broad `pull_request` and `workflow_dispatch` triggers with no narrow path filters, so schema, routing, runtime, and role-file changes are not skipped by path selection.
- F-3 addressed: the workflow avoids `pull_request_target`, secrets, and write permissions; token scope is `contents: read`.
