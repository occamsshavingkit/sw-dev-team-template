## 3. Implementation sketch

- Create `.github/ISSUE_TEMPLATE/framework-gap.yml` and its parent directory if absent.
- Use GitHub issue form fields from `docs/prior-art/T044.md`: markdown intro, template version, gap category, affected file/section, abstract downstream context, observed outcome, why it is a framework gap, suggested fix, redaction confirmation, and opt-in confirmation.
- Set required fields for version, category, location, observed outcome, gap rationale, redaction confirmation, and opt-in confirmation.
- Avoid fields that solicit customer name, downstream repository name, product codename, or private project identifiers.
- Align category dropdown values with T043 labels if T043 lands first; otherwise use the same proposed category names and reconcile during review.

## 8. Duel

### 8.1 Findings (`qa-engineer`)

- **F-1 [Critical]:** If required checkboxes for redaction and opt-in are optional, ambiguously worded, or placed only in markdown help text, users can submit framework-gap reports without explicit consent to upstream filing; the failure is a valid-looking issue that violates the filing contract.
- **F-2 [Major]:** If the category dropdown drifts from T043 labels or uses display names that cannot be mapped back to labels, T044 creates a second taxonomy authority; the failure is inconsistent triage between form submissions and label-based reporting.
- **F-3 [Major]:** If the form uses unsupported GitHub issue-form keys, invalid YAML indentation, or invalid checkbox syntax, GitHub can ignore or hide the template; the failure is loss of structured intake with users falling back to unvalidated free-form issues.

### 8.2 Rebuttals / revisions (`software-engineer`)

- F-1 addressed: `.github/ISSUE_TEMPLATE/framework-gap.yml` requires explicit redaction and opt-in checkboxes with required options, not markdown-only guidance.
- F-2 addressed: the category dropdown uses the same `framework:*` labels defined in T043's taxonomy.
- F-3 addressed: the form uses basic GitHub issue-form keys only (`name`, `description`, `title`, `labels`, `body`, `type`, `id`, `attributes`, `validations`) and is YAML-parse validated.
