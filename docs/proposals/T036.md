## 3. Implementation sketch

- **Module / file set touched:** create `schemas/agent-contract.schema.json`; do not edit `.claude/agents/*.md` in this task.
- **Public interfaces introduced or modified:** new JSON Schema 2020-12 validation contract for extracted agent metadata consumed later by `scripts/lint-agent-contracts.sh`.
- **Key internal data shapes:** an agent record with `name`, `description`, `tools`, `model`, `source_path`, `local_supplement_pattern`, `role_authority`, `customer_interface`, `escalation`, `output`, and `workflow_pipeline` fields where available from canonical Markdown/frontmatter.
- **Schema boundaries:** require stable frontmatter fields (`name`, `description`, `tools`, `model`) and validate extracted policy flags without trying to encode full prose semantics.
- **Control flow in broad strokes:** T039 extracts Markdown/frontmatter into JSON records, validates records against this schema with draft 2020-12, and reports path-scoped errors.
- **Authority rule:** the schema validates projections of canonical role files; it does not become a role contract or replacement source of truth.

- **External services / APIs:** none.

## 5. Test plan outline

## 8. Duel

**Solution Duel** — pre-code adversarial review. `qa-engineer` writes
three concrete ways this proposal could fail before implementation.

### 8.1 Findings (`qa-engineer`)

- **F-1:** Schema accepts runtime/generated agent wrappers as canonical roles — **How it fails:** T039 validates OpenCode/runtime adapter output as role authority because `source_path` or `role_authority` is not constrained to canonical `.claude/agents/*.md` inputs. — **Severity if unaddressed:** blocker.
- **F-2:** Required policy flags are optional when prose extraction misses them — **How it fails:** a role file that drops customer-interface or escalation constraints still passes because only frontmatter fields are required. — **Severity if unaddressed:** major.
- **F-3:** Tool declarations are under-specified — **How it fails:** malformed `tools` values such as a single comma-separated string, unknown tool names, or empty entries pass validation and later lint cannot detect role/tool drift. — **Severity if unaddressed:** major.

### 8.2 Rebuttals / revisions (`software-engineer`)

- **F-1 addressed:** `schemas/agent-contract.schema.json` constrains `source_path` to `.claude/agents/*.md`, requires `role_authority.canonical_source = canonical-role-file`, and requires `role_authority.generated_artifact = false`, so runtime/generated wrappers cannot validate as canonical role contracts.
- **F-2 addressed:** `customer_interface`, `escalation`, `output`, and `workflow_pipeline` are required objects with non-empty evidence where applicable; T039 extraction fails closed on missing frontmatter and validates these policy projections.
- **F-3 addressed:** `tools` is a required non-empty unique array whose items are limited to the supported tool-name enum; malformed or unknown tool declarations fail validation.
