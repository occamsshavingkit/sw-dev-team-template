## 3. Implementation sketch

- **Module / file set touched:** create `scripts/lint-agent-contracts.sh`; consume `schemas/agent-contract.schema.json`, `schemas/model-routing.schema.json`, and `schemas/generated-artifact.schema.json` when present.
- **Public interfaces introduced or modified:** new repo-root command `scripts/lint-agent-contracts.sh` with deterministic output and non-zero exit for G6-blocking validation failures.
- **Key internal data shapes:** temporary normalized JSON records for agent contracts, model-routing samples or fallback events, and generated-artifact manifests.
- **Control flow:** verify repo root, locate required schema files, validate schemas as draft 2020-12, extract frontmatter from `.claude/agents/*.md`, validate extracted records, validate generated-artifact manifests if T040 has produced them, print file-scoped failures, and exit non-zero on errors.
- **Validator strategy:** use a pinned runtime validator command, defaulting to `npx --yes ajv-cli@5.0.0 validate --spec=draft2020`; allow an `AJV_BIN` override for preinstalled CI/local tooling, and fail with a clear missing-validator message rather than silently passing.
- **Non-goal:** do not edit canonical agent files, generated runtime files, or schemas during lint.

## 6. Risks and mitigations

| Risk | Mitigation |
|---|---|
| Lint passes because validator is missing. | Treat missing validator as a clear failure for this strict gate. |
| Markdown/frontmatter parsing is too brittle. | Extract only simple known frontmatter keys and fail with path-scoped messages. |
| Script writes generated outputs during validation. | Keep lint read-only except temporary files under `mktemp`. |
| Ajv draft mismatch causes false results. | Pin `--spec=draft2020` in every Ajv CLI invocation. |
| Runtime `npx` introduces unpinned or hidden dependency drift. | Pin `ajv-cli@5.0.0`, expose `AJV_BIN` for reproducible CI installs, and do not add root package metadata unless a later ADR makes committed tooling mandatory. |

## 8. Duel

**Solution Duel** — pre-code adversarial review. `qa-engineer` writes
three concrete ways this proposal could fail before implementation.

### 8.1 Findings (`qa-engineer`)

- **F-1:** Lint silently ignores missing schemas — **How it fails:** the script treats absent T036-T038 schema files as optional and exits zero, so G6 evidence reports contract validation before the contract exists. — **Severity if unaddressed:** blocker.
- **F-2:** Temporary extraction masks frontmatter parse failures — **How it fails:** malformed YAML or missing delimiters produce partial JSON records that still validate, hiding broken role contracts behind defaulted fields. — **Severity if unaddressed:** major.
- **F-3:** `npx --yes` runtime fetch breaks deterministic gates — **How it fails:** CI without network access or with npm registry changes fails for infrastructure reasons instead of contract defects, blocking releases unpredictably. — **Severity if unaddressed:** major.

### 8.2 Rebuttals / revisions (`software-engineer`)

- **F-1 addressed:** `scripts/lint-agent-contracts.sh` treats all three schema files as required and exits non-zero when any is absent.
- **F-2 addressed:** frontmatter extraction fails closed on missing delimiters, missing required keys, filename/name mismatches except for the explicit SME template case, empty tool lists, and unknown tools.
- **F-3 addressed:** the default validator remains the required pinned runtime `npx --yes ajv-cli@5.0.0`, while `AJV_BIN` allows CI to use a preinstalled validator path for networkless deterministic gates.
