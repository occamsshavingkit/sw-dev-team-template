## 3. Implementation sketch

- **Module / file set touched:** `docs/TEMPLATE_UPGRADE.md` only.
- **Public interfaces introduced or modified:** none; this documents operator retrofit procedure.
- **Key internal data shapes:** downstream root `ROADMAP.md`, project-owned roadmap, template-release leakage, and `.template-customizations` preservation when a roadmap is intentional.
- **Control flow in guidance:** add a short retrofit rule near scaffold/upgrade handling; tell maintainers to classify an existing downstream root `ROADMAP.md` before changing it.
- **Classification outcomes:** preserve if product-owned or intentionally customized; remove or replace if it is copied template release-plan leakage; record the classification as downstream-reference evidence.
- **Non-goal:** do not automate deletion or add a migration script because `docs/prior-art/T026.md` identifies semantic classification as required.

## 8. Duel

**Solution Duel** — pre-code adversarial review. `qa-engineer` writes
three concrete ways this proposal could fail before implementation.

### 8.1 Findings (`qa-engineer`)

- **Critical:** If the retrofit rule lacks an explicit ambiguous-case path
  such as stop, preserve, and request review, maintainers may silently
  remove a downstream product roadmap they cannot classify.
- **Major:** If preserved downstream roadmaps are not connected to
  `.template-customizations` or equivalent evidence, later upgrade runs can
  keep flagging the same intentional file as leakage and create repeated
  churn.
- **Major:** If "remove or replace" is not constrained to template
  release-plan leakage, operators may normalize replacing product roadmap
  content with framework wording instead of preserving project-owned truth.

### 8.2 Rebuttals / revisions (`software-engineer`)

- Addressed: `docs/TEMPLATE_UPGRADE.md` now says ambiguous downstream
  `ROADMAP.md` ownership must stop, preserve, and request review.
- Addressed: preserved intentional roadmaps are tied to project-owned
  handling and `.template-customizations` when upgrade tooling would keep
  flagging the path.
- Addressed: remove or replace applies only to copied template release-
  plan leakage, not product roadmap content.
