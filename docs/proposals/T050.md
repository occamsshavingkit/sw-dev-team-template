# Proposal — T050

## 1. Task reference

- **Task ID:** T050
- **One-line task statement:** Enforce the M7 patch-size limit in `docs/workflow-pipeline.md` and `.github/workflows/improve-template.yml`, including the maximum changed-file and diff-line threshold used to block oversized automation output.
- **Trigger clauses that fire:** 3 — cross-module boundary.
- **Prior-art reference:** `docs/prior-art/T050.md`.
- **ADR reference:** none.

## 2. Chosen ADR path

- **Alternative chosen:** none — no ADR.
- **Reason:** The task defines and enforces policy constants; no ADR exists, and prior art says the numeric thresholds are project policy.

## 3. Implementation sketch

- Touch `docs/workflow-pipeline.md` and `.github/workflows/improve-template.yml`.
- Define explicit constants in documentation: `M7_MAX_CHANGED_FILES=5` and `M7_MAX_DIFF_LINES=400` for automated M7 output.
- Implement the same constants in `improve-template.yml` before PR creation where possible.
- Compute changed files and added/deleted diff lines from Git against the base branch, not from GitHub path filters alone.
- If either limit is exceeded, block PR creation or update the no-op/failure output with measured totals and the selected issue/gap.
- Count every changed tracked file, including generated files, docs, workflow files, renames, and deletions; no path exclusions in the initial policy.
- Count diff lines as added plus deleted text lines from `git diff --numstat` against the base branch; binary files count as one changed file and zero diff lines unless Git reports numeric additions/deletions.

## 4. Dependencies touched

- No package dependencies.
- Internal dependencies: `docs/workflow-pipeline.md`, `.github/workflows/improve-template.yml` from T049.
- External service: Git and GitHub Actions runner environment, per `docs/prior-art/T050.md`.

## 5. Test plan outline

- Unit: shell syntax validation for patch-size check.
- Integration: local diff scenarios under and over file/line limits.
- System / acceptance: dry-run workflow proves oversized output fails before PR creation or reports failure without broken commit.
- Performance / security: ensure check does not rely on untrusted PR-provided commands and handles filenames safely.

## 6. Risks and mitigations

| Risk | Mitigation |
|---|---|
| Numeric limits are arbitrary. | Start conservative, document as M7 policy, and let maintainers adjust in one documented place. |
| Documentation and workflow constants drift. | Name constants identically and include a validation note in review. |
| Diff counting mishandles generated files or renames. | Use Git's own changed-file and numstat output and document inclusion rules. |

## 7. Open questions

- None for code start. Decision: use `5` changed files and `400` added-plus-deleted diff lines as the initial M7 automation limits.

## 8. Duel

### 8.1 Findings (`qa-engineer`)

- **F-1 [Major]:** If the workflow and documentation each define `M7_MAX_CHANGED_FILES` and `M7_MAX_DIFF_LINES` independently, future edits can drift; the failure is automation accepting a patch size that the binding workflow policy says must be blocked, or vice versa.
- **F-2 [Major]:** If the diff check uses GitHub path filters or event payload file lists instead of `git diff` against the actual base branch, large generated or renamed changes can be missed; the failure is an oversized automation PR passing the size gate.
- **F-3 [Critical]:** If filenames from `git diff --numstat` are parsed with whitespace-delimited shell loops, paths containing spaces, tabs, renames, or leading dashes can corrupt counts or execute as options; the failure is a bypassable size limit with unsafe shell behavior.

### 8.2 Rebuttals / revisions (`software-engineer`)

- F-1 addressed: `docs/workflow-pipeline.md` and `.github/workflows/improve-template.yml` use identical constant names and values: `M7_MAX_CHANGED_FILES=5` and `M7_MAX_DIFF_LINES=400`.
- F-2 addressed: the workflow computes changed files and diff lines from `git diff` against the checked-out branch, not from GitHub path filters or event payload file lists.
- F-3 addressed: changed-file counts use `git diff --name-only` piped to `wc -l`, and diff-line counts use `git diff --numstat` parsed by `awk` over numeric columns only; filenames are not shell-expanded or executed.
