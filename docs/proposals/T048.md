## 8. Duel

### 8.1 Findings (`qa-engineer`)

- **F-1 [Major]:** If the smoke command set is not enumerated before implementation, the workflow can duplicate T046 while omitting runtime compilation or generated-drift checks; the failure is a green smoke gate that never proves template contracts are executable.
- **F-2 [Critical]:** If generation runs before a clean-tree check or uses write permissions to push regenerated outputs, CI can mutate canonical artifacts instead of reporting drift; the failure is hidden generator instability landing as automated churn.
- **F-3 [Major]:** If missing scripts are treated as optional skips to keep smoke fast, fresh clones or downstream scaffolds can pass without the very contract tools they need; the failure is template breakage discovered only after adoption.

### 8.2 Rebuttals / revisions (`software-engineer`)

- F-1 addressed: `.github/workflows/template-contract-smoke.yml` enumerates contract lint, runtime compilation, and generated-output drift detection.
- F-2 addressed: the workflow has read-only repository permission and fails on generated drift with `git diff` instead of committing regenerated artifacts.
- F-3 addressed: required scripts are checked with `test -x`; missing scripts fail loudly rather than being skipped.
