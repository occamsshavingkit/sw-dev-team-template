## 5. Known pitfalls

- Vague expected behavior: regression cases must assert concrete preserved behaviors, not subjective prompt quality.
- Golden-answer brittleness: for LLM prompts, pass/fail should focus on invariant preservation and forbidden behaviors, not exact prose.
- Authority drift: tests must compare runtime/generated outputs to canonical role files and hard rules, not to previous generated output alone.

## 6. Recommendation to `architect` / `software-engineer`

For `docs/prompt-regression.md`, reuse the M0/M1 scenario shape and cover at least `tech-lead`, `researcher`, `software-engineer`, `qa-engineer`, `code-reviewer`, and generated-artifact/runtime-compiler behavior; make each case list stimulus, expected invariants, forbidden behaviors, evidence path, and pass/fail criteria.
