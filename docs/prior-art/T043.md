## 5. Known pitfalls

- Repository-scoped labels do not propagate across repositories automatically; downstream repositories need their own label definitions or documented mapping.
- Default labels are generic; framework-specific gaps need explicit names and descriptions to avoid mixing template defects with product bugs.
- `docs/ISSUE_FILING.md` includes redaction and opt-in requirements; taxonomy text must preserve those filing constraints.

## 6. Recommendation to `architect` / `software-engineer`

Treat GitHub labels as the canonical tracking primitive and keep the taxonomy in `docs/ISSUE_FILING.md`; include categories for routing, rule defect, template gap, standards gap, missing dependency/tooling, downstream leakage, and release-policy gap, with labels mapped to the existing issue-filing fields.
