## 5. Known pitfalls

- `workflow_dispatch` only receives events when the workflow file exists on the default branch.
- Scheduled workflows run on the latest default-branch commit; uncontrolled schedules can drift from the intended one-improvement-at-a-time model.
- Automation must not collapse multiple framework gaps into one broad PR, or it defeats the patch-size and human-review constraints.

## 6. Recommendation to `architect` / `software-engineer`

Define the M7 loop as one issue or selected gap per run, with a single proposal, one branch, one PR, required checks, patch-size enforcement, and human review before merge; failures should produce no-op status or an issue comment/update, not direct commits.
